1. environment.yaml
Path:

/sswg_skeleton/reproducibility/environment.yaml

Purpose:
Canonical Conda environment definition for the entire MVM.
This is the “source of truth” for Python version, major dependencies, and reproducibility across systems.

────────────────────────────────────────

2. flake.nix

Path:

/sswg_skeleton/reproducibility/flake.nix

Purpose:
Nix-based reproducible environment.
Complements Conda by enabling fully deterministic builds for NixOS/downstream systems.

────────────────────────────────────────

3. requirements.in

Path:

/sswg_skeleton/reproducibility/requirements.in

Purpose:
Primary dependency specification for pip-tools (pip-compile).

────────────────────────────────────────

4. requirements.lock

Path:

/sswg_skeleton/reproducibility/requirements.lock

Purpose:
Fully pinned, deterministic dependency lock file generated by pip-tools.
Used to ensure absolute version reproducibility.

────────────────────────────────────────

5. requirements.txt

Path:

/sswg_skeleton/reproducibility/requirements.txt


Purpose:
Autogenerated installation file based on the locked versions.
Users run:

pip install -r reproducibility/requirements.txt


────────────────────────────────────────

6. settings.json

Path:

/sswg_skeleton/reproducibility/settings.json


Purpose:
Repository-wide VSCode configuration overrides.
This is not the workspace .vscode/settings.json — this is a canonical template.
Later, the CLI can generate .vscode/settings.json from this file using:

sswg configure editor


────────────────────────────────────────

7. vscode_extensions.json

Path:

/sswg_skeleton/reproducibility/vscode_extensions.json

Purpose:
Defines the authoritative list of recommended or required VSCode extensions.
These are injected into .vscode/extensions.json via CLI automation.
